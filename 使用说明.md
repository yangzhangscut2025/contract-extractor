# 合同智能提取工具 — 使用说明

**版本**：V1.7 | **日期**：2026-06-19

---

## 一、简介

批量处理 PDF 合同文件，自动提取 43 个字段（姓名、薪酬、银行账户等），支持人工校对、翻译辅助，导出 Excel。

适用：劳动合同、调薪文件（PDF 格式，文字型/扫描件均可）。

---

## 二、安装与启动

- Windows 10 / 11
- 解压后双击 `启动.bat`

---

## 三、首次配置

左侧菜单 **设置**。

### 3.1 大模型（必填）

| 配置项 | 说明 |
|--------|------|
| 服务商 | DeepSeek 或智谱 AI |
| 模型名称 | 默认 `deepseek-chat` |
| API Key | 服务商密钥 |

填写后点击 **测试连接**。

> DeepSeek：https://platform.deepseek.com → API Keys
> 智谱 AI：https://open.bigmodel.cn → API Keys

### 3.2 OCR（可选）

仅扫描件需要，文字型 PDF 不受影响。

| 配置项 | 说明 |
|--------|------|
| AccessKey ID | 阿里云 RAM 用户 AccessKey |
| AccessKey Secret | 对应 Secret |

---

## 四、文件管理

点击 **文件管理**，拖拽或点击导入 PDF。重复文件自动识别跳过，提示准确数量。

### 4.1 筛选

两个下拉框可组合使用：

| 筛选 | 选项 |
|------|------|
| 校验状态 | 全部 / 已校验 / 未校验 |
| 处理状态 | 全部 / 待处理 / 已处理 / 失败 |

全选勾选框只勾选当前筛选结果。

### 4.2 列表列

| 列 | 说明 |
|----|------|
| 序号 | 与员工序号表 CSV 对应 |
| 文件名 | 悬停显示完整路径 |
| 员工编号 | 从文件名自动解析 |
| 合同编号 | 处理时生成 |
| 状态 | 待处理/处理中/已完成/OCR失败/失败 |
| 已验证 | 是否已人工校对 |
| 错误信息 | 失败原因 |

### 4.3 按钮

| 按钮 | 说明 |
|------|------|
| 原件 | 打开所在文件夹并定位文件 |
| 校对 | 进入校对审核页（所有状态可用） |
| 删除 | 删除记录及提取结果 |
| 开始处理 | 处理选中文件，未选则处理全部待处理 |
| 取消处理 | 中断处理 |
| 清理重复 | MD5 去重，保留字段最多的 |

## 五、处理流程

```
pdfjs 提取文字
  ├─ 文字足够 → 直接用（免费）
  ├─ 文字不足 + OCR 已配 → 调 OCR
  └─ 文字为空 + OCR 未配 → 失败
→ 文件分类 → 合同编号 → LLM 提取 43 字段 → 校验
```

OCR 只在 pdfjs 提取文字不足时才触发，大部分 PDF 不花钱。

## 六、校对审核

点击 **校对审核** 或文件列表点 **校对**。

### 6.1 界面

| 区域 | 内容 |
|------|------|
| 左侧 | PDF 原版（上下滑动），支持翻译切换 |
| 右侧 | 43 个字段表格 |

### 6.2 按钮

| 按钮 | 说明 |
|------|------|
| 重提 | 重新提取，手动修正值不丢失 |
| 翻译 | LLM 译为中文，日期/金额/人名保留原文，缓存不重复调用 |
| PDF 原版 / 中文翻译 | 切换左侧视图 |
| 标记为已验证 | 确认校对完成 |

### 6.3 字段表格

| 列 | 说明 |
|----|------|
| 字段名 | 点击高亮定位 |
| 提取值 | LLM 结果 |
| 修正值 | 点击编辑 |
| 校验 | 绿勾=通过，黄叹=可疑 |

### 6.4 字段高亮

点击字段名 → 左侧 PDF 黄色高亮匹配文字，支持日期/金额/电话多格式智能搜索。

## 七、导出

| 按钮 | 说明 |
|------|------|
| 导出选中 | 勾选的文件 |
| 导出全部已处理 | 已完成 + 失败 + OCR 失败 |
| 导出失败 (N) | 仅失败 + OCR 失败 |

Excel 包含 48 列（文件名称 + 员工编号 + 合同编号 + 合同类别 + 43 字段 + 错误信息）。校验异常黄色标注，失败文件自动填入员工编号和合同编号。

## 八、常见问题

**Q：字段为空？**
原文中确实没有则留空，可手动补充。

**Q：扫描件失败？**
检查 OCR 配置，或确认是否为文字型 PDF。

**Q：调薪文件字段少？**
调薪文件通常只含薪酬变更，正常。

**Q：翻译是旧数据？**
点重提清缓存后重新翻译。

**Q：OCR 失败状态橙色？**
OCR 失败但 pdfjs 提取到了文字，不影响使用。

**Q：文件重复？**
点清理重复按钮。

**Q：API Key 测试失败？**
重新粘贴确保无多余空格/换行。
