# 合同智能提取工具 — 使用说明

**版本**：V1.7 | **日期**：2026-06-18

---

## 一、简介

本工具用于批量处理 PDF 合同文件，自动提取 46 个字段（姓名、薪酬、银行账户等），支持人工校对、翻译辅助，最终导出为 Excel。

适用文件：劳动合同、调薪文件（PDF 格式，文字型/扫描件均可）。

---

## 二、安装与启动

**环境要求**：Windows 10 / 11

**启动**：解压后双击 `启动.bat`。

---

## 三、首次配置

点击左侧菜单 **设置**。

### 3.1 大模型（必填）

| 配置项 | 说明 |
|--------|------|
| 服务商 | DeepSeek 或智谱 AI |
| 模型名称 | 默认 `deepseek-chat` |
| API Key | 服务商提供的密钥 |

**获取 API Key**：
- DeepSeek：https://platform.deepseek.com → API Keys
- 智谱 AI：https://open.bigmodel.cn → API Keys

填写后点击 **测试连接**，确认通过。

### 3.2 OCR（可选）

仅扫描件 PDF 需要。不配也能处理文字型 PDF。

| 配置项 | 说明 |
|--------|------|
| AccessKey ID | 阿里云 RAM 用户 AccessKey |
| AccessKey Secret | 对应的 Secret |
| 区域 | 默认 `cn-hangzhou` |

---

## 四、文件导入

点击 **文件管理**。

- **拖拽导入**：将 PDF 拖入虚线框
- **点击导入**：点击虚线框下方链接选择文件
- 重复文件自动识别并跳过

文件列表各列说明：

| 列 | 说明 |
|----|------|
| 序号 | 与员工序号表 CSV 对应 |
| 文件名 | 悬停显示完整路径 |
| 员工编号 | 自动从文件名解析 |
| 合同编号 | 处理时自动生成 |
| 状态 | 见下方 |
| 错误信息 | 失败原因 |
| 原件 | 打开所在文件夹 |

**状态说明**：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 待处理 | 灰 | 已导入，未提取 |
| 处理中 | 蓝 | 正在提取 |
| 已完成 | 绿 | 提取完成 |
| OCR失败 | 橙 | OCR 失败，使用 PDF 文字层 |
| 失败 | 红 | 处理出错 |
| 已跳过 | 橙 | 无法识别（极少出现） |

---

## 五、批量处理

1. 勾选文件（或点 **开始处理** 自动处理所有待处理文件）
2. 顶部显示进度
3. 每次最多 3 个文件并发，失败自动重试 1 次

加密 PDF 会自动弹窗要求输入密码。

---

## 六、校对审核

点击 **校对审核**，或文件列表中点 **校对**。

### 6.1 界面

| 区域 | 内容 |
|------|------|
| 左侧 | PDF 原版（上下滑动翻页），支持翻译切换 |
| 右侧 | 字段表格 |

### 6.2 字段表格

| 列 | 说明 |
|----|------|
| 字段名 | 点击可高亮定位 |
| 提取值 | LLM 自动提取 |
| 修正值 | 点击可直接编辑 |
| 校验 | 绿勾=通过，黄叹=可疑 |

### 6.3 字段高亮

点击字段名 → 左侧 PDF 自动黄色高亮匹配文字，支持日期/金额/电话多格式智能搜索。

### 6.4 翻译

点击 **翻译** 按钮，LLM 将原文译为中文。日期、金额、人名、邮箱等关键信息保留原文。翻译结果缓存，同文件不重复调用。

点击顶部 **PDF 原版 / 中文翻译** 切换视图。

### 6.5 重提与标记

- **重提**：重新提取，手动修正值不丢失
- **标记已验证**：确认校对完成

---

## 七、导出

点击 **导出**，三个按钮：

| 按钮 | 说明 |
|------|------|
| 导出选中 | 勾选的文件 |
| 导出全部已处理 | 已完成 + 失败 + OCR失败 |
| 导出失败 | 仅失败 + OCR失败 |

Excel 包含 46 列，校验异常的单元格以黄色标注。失败文件的员工编号和合同编号自动填入。

## 八、常见问题

**Q1：字段为空？**
原文中确实没有该信息则留空，可在校对页手动补充。

**Q2：扫描件处理失败？**
配置阿里云 OCR 或确认文件是否为文字型 PDF。

**Q3：调薪文件很多字段为空？**
调薪文件通常只含薪酬变更，其他信息原文不存在，属正常。

**Q4：PDF 加密？**
处理时弹窗输入密码。

**Q5：API Key 在哪获取？**
见第三章链接。

**Q6：OCR 失败但字段正常？**
文件是文字型 PDF，OCR 未使用但 pdfjs 成功提取了文字，标绿色正常。

**Q7：翻译是旧数据？**
点"重提"后再点"翻译"即可更新。

**Q8：文件重复？**
点文件管理页的 **清理重复** 按钮。
