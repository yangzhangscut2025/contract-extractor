# 合同智能提取工具 — 使用说明

**版本：V1.7  |  更新日期：2026-06-17**

---

## 一、软件概述

本工具用于**批量处理 PDF 合同文件**，自动提取员工信息、薪酬、银行账户等 45 个关键字段，支持人工校对后导出为 Excel。

**适用文件**：劳动合同、调薪文件（PDF 格式，支持文字型和扫描件）

---

## 二、安装与启动

### 环境要求
- Windows 10 / 11 操作系统
- 无需额外安装运行环境（软件自带）

### 启动步骤
1. 双击程序图标启动
2. 等待主界面加载完成

---

## 三、首次使用 — 配置

首次使用需要配置**大模型 API Key**（必须）和**阿里云 OCR**（可选，仅扫描件需要）。

### 3.1 大模型配置

点击左侧菜单 **设置**，在"大模型配置"卡片中填写：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 大模型服务商 | 选择 DeepSeek 或 智谱 AI | DeepSeek |
| 模型名称 | 保持默认或填入模型代号 | `deepseek-chat` |
| API Key | 服务商提供的密钥 | `sk-xxxxxxxx` |

> **获取 API Key**：
> - DeepSeek：访问 https://platform.deepseek.com → 注册 → API Keys → 创建
> - 智谱 AI：访问 https://open.bigmodel.cn → 注册 → API Keys → 创建

填写后点击 **测试连接**，显示"连接测试成功"即配置完成。点击 **保存**。

### 3.2 阿里云 OCR 配置（可选）

如果处理的是**扫描件 PDF**（文字层为空，需要 OCR 识别），需要配置阿里云 OCR：

| 配置项 | 说明 |
|--------|------|
| AccessKey ID | 阿里云 RAM 用户的 AccessKey |
| AccessKey Secret | 对应的 Secret |
| 区域 | 默认 `cn-hangzhou` |

> **获取 AccessKey**：登录阿里云控制台 → RAM 访问控制 → 用户 → 创建用户 → 勾选"OpenAPI 调用访问" → 获取 AccessKey ID/Secret → 为该用户授权 `AliyunOCRFullAccess` 权限。

填写后点击 **测试连接**，点击 **保存**。

## 四、文件导入

点击左侧菜单 **文件管理**。

### 4.1 导入方式

- **拖拽导入**：将 PDF 文件从文件夹直接拖入虚线框
- **点击导入**：点击虚线框下方"或点击此处选择文件"

导入成功后，文件出现在下方列表中。**重复文件会自动识别并跳过**。

### 4.2 文件列表

| 列 | 说明 |
|----|------|
| 文件名 | PDF 文件名 |
| 员工编号 | 自动从文件名解析（取第一个 `__` 左侧） |
| 合同编号 | 处理时自动生成（格式：`员工编号_序号`） |
| 合同类别 | 劳动合同 / 调薪文件 / 其他 |
| 状态 | 见下方 |
| 已验证 | 是否已人工校对确认 |

**状态说明**：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 待处理 | 灰色 | 已导入，尚未提取 |
| 处理中 | 蓝色 | 正在提取字段 |
| 已完成 | 绿色 | 提取完成，可校对 |
| 失败 | 红色 | 处理出错，可查看错误信息 |
| 已跳过 | 橙色 | 文件类型无法识别（非合同），已自动跳过 |

### 4.3 原件审查

点击文件右侧的 **原件** 按钮，用系统默认 PDF 阅读器打开原始合同文件。

## 五、批量处理

### 5.1 开始处理

1. 勾选需要处理的文件（或点击 **开始处理** 自动处理所有"待处理"文件）
2. 点击 **开始处理** 按钮
3. 顶部显示处理进度（完成数 / 失败数 / 当前步骤）

每次最多 **3 个文件同时处理**，失败会自动重试 1 次。

### 5.2 加密 PDF

遇到加密 PDF 时，会自动弹出密码输入框。输入密码后点击 **确定**，错误则提示重试。不知道密码可点击 **跳过**，该文件标记为失败。

---

## 六、校对审核

点击左侧菜单 **校对审核**，或点击文件列表中的 **校对** 按钮。

### 6.1 界面说明

| 区域 | 内容 |
|------|------|
| 左侧 | PDF 原版文件（上下滑动翻页） |
| 右侧 | 提取字段表 |

### 6.2 查看原文

- **上下滑动**浏览 PDF 所有页面
- 左侧搜索框输入关键词 → 回车，在数据库原文中搜索，显示匹配行号

### 6.3 字段校对

字段表格包含四列：

| 列 | 说明 |
|----|------|
| 字段名 | 点击可从原文中搜索定位 |
| 提取值 | LLM 自动提取的结果 |
| 修正值 | 点击可手动编辑（回车保存） |
| 校验 | 绿勾 = 通过、黄叹号 = 可疑（待核对）、红叉 = 错误 |

### 6.4 翻译辅助核对

对于非中文合同，可点击文件选择器右侧的 **翻译** 按钮（🌐 图标），将原文翻译为中文。

- 翻译结果**缓存数据库**，同一文件不会重复翻译
- 点击顶部的 **PDF 原版 / 中文翻译** 切换视图
- 关键信息（日期、金额、人名、邮箱、电话等）**保留原文不翻**，方便对照
- 审阅者读中文翻译，交叉验证右侧字段表提取是否准确

### 6.5 字段高亮定位

点击字段名或字段值，左侧 PDF 会自动**黄色高亮**对应文字，并滚动到第一处匹配位置。

> 支持智能搜索：日期、金额、电话、姓名会自动生成多种格式变体搜索，减少因格式差异导致漏匹配。

### 6.6 标记已验证

校对完成后，点击 **标记为已验证**，文件列表中该文件会显示绿色"是"标签。

### 6.7 重新提取

如果 Prompt 更新后想用新规则重新提取某个文件，在文件选择器右侧点击 **🔄 重提** 按钮即可。

## 七、导出 Excel

点击左侧菜单 **导出**。

- **导出选中文件**：导出在文件列表中勾选的文件
- **导出全部已完成**：导出所有状态为"已完成"的文件

选择保存路径后自动生成 Excel 文件，包含 45 列完整数据。

### Excel 列的颜色含义

| 背景色 | 含义 |
|--------|------|
| 无颜色 | 正常通过 |
| 黄色 | 字段值可疑，建议人工核对 |
| 蓝色表头 | 列名 |
| 灰色隔行 | 便于阅读 |

## 八、常见问题

### Q1：为什么有些字段是空的？
LLM 从 PDF 原文中找到对应信息才会填入。如果原文中确实没有该信息（如没有 IBAN），字段留空是正常的。请在校对面板中人工补充。

### Q2：扫描件处理不了怎么办？
确认为扫描件（文字层为空）时，需要：
1. 在设置页配置阿里云 OCR AccessKey
2. 重新处理该文件，工具会自动调用 OCR 识别

### Q3：调薪文件的字段为什么是空的？
调薪文件和劳动合同使用同一套 42 字段提取。调薪文件通常只包含薪酬变更信息，其他不相关的字段（如个人信息）在原文中不存在，自然会留空。如需对这些字段补充，可在校对面板手动编辑，或点击 **重新提取** 按钮重试。

### Q4：PDF 加密了怎么办？
处理到加密 PDF 时，会自动弹出密码输入框，输入密码即可。如果不清楚密码，点击"跳过"。

### Q5：API Key 在哪里获取？
- DeepSeek：https://platform.deepseek.com → 注册 → API Keys
- 智谱 AI：https://open.bigmodel.cn → 注册 → API Keys
- 阿里云 OCR：阿里云控制台 → RAM 访问控制 → 创建 AccessKey

### Q6：处理失败的文件怎么处理？
点击 **原件** 按钮查看原文件，确认文件是否损坏、是否为支持的合同类型。然后在文件管理页面重新勾选 → **开始处理** 重试。
