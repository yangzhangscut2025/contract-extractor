合同智能提取工具 — 产品需求文档


版本：V1.7 | 日期：2026-06-18 | 作者：张阳 | 用户：HR 运营 / 数据校验
---

1. 概述


1.1 背景


公司收到来自各国的劳动合同及调薪文件（PDF），格式不统一、语言多样，人工录入效率低且易出错。文件名有规律，但文件内不含合同编号，需工具自动生成。

1.2 目标


开发一款 Electron 桌面工具，实现：
  • 批量导入 PDF，自动识别文件类型
  • 提取 43 个字段（LLM）+ 3 个系统生成字段 = 46 列
  • 自动生成合同编号（同员工递增）
  • 文字型 PDF 直接提取，扫描件调用阿里云 OCR
  • 人工校对、翻译辅助、Excel 导出

1.3 适用范围


  • 输入：系统导出的 PDF，文件名格式固定
  • 类型：劳动合同、调薪文件
  • 语言：英文、泰文、中文等多语言混合

---

2. 业务流程


2.1 文件命名规则


  • 示例：CE3E5PLL__1.Mr.Chanon - Employment_agreement.pdf
  • 规则：取第一个 __ 左侧为员工编号（如 CE3E5PLL）
  • 容错：无 __ 时取前 8 字符

2.2 处理流程


导入 PDF
  → 文件名解析（员工编号）
  → 文本提取（OCR 优先，失败回退 pdfjs）
  → 文件分类（关键词 + LLM，不确定默认劳动合同）
  → 合同编号生成（员工编号_序号，递增不填补）
  → LLM 提取 43 字段（输出 JSON）
  → 补漏校验（日期格式、邮箱格式等）
  → 展示校对（左侧 PDF 原版，右侧字段表格）
  → 导出 Excel（46 列）

---

3. 功能详述


3.1 文件管理


功能说明
导入支持拖拽/点击，批量选择 PDF，MD5 自动去重
列表显示序号、文件名、员工编号、合同编号、合同类别、状态、错误信息
排序按员工编号升序，与员工序号表 CSV 对应
清理重复一键去重，保留提取字段最多的记录
原件打开文件所在文件夹并定位

3.2 文件分类


1. 关键词匹配（前 2000 字符）：employment agreement、salary adjustment、劳动合同 等 2. 规则无法确定 → LLM 快速分类 3. 仍不确定 → 默认按劳动合同处理

3.3 文本提取


场景处理方式
OCR 已配置PDF 逐页渲染为图片 → 阿里云 OCR → 拼接文本
OCR 未配置pdfjs 提取文字层
OCR 失败回退 pdfjs；文字量足够则标 `completed`，否则标 `ocr_failed`
PDF 加密弹出密码输入框

3.4 合同编号


  • 格式：员工编号_序号（如 CE3E5PLL_01）
  • 序号 = 该员工历史最大序号 + 1，删除不填补
  • 同一员工多文件按导入顺序分配

3.5 字段提取


劳动合同与调薪文件统一使用 43 字段 Prompt（见附录 A），LLM 自动从原文提取存在的字段，缺失为 null。
字段分类：
类别包含字段
基础信息姓名、性别、国籍、生日、年龄、婚姻状况、出身地
雇佣信息雇主名称、就职国家、岗位名称、入职时间
合同期限期限类型、期限、单位、生效/结束日期、试用期
薪酬年薪/月薪/时薪及币种、交通补贴、餐补、奖金、其他补贴
休假年假天数、病假天数
银行信息银行名称、地址、账户名、账号、IBAN、SWIFT
联系方式个人邮箱、工作邮箱、电话、地址、身份证件、社保号
系统生成员工编号、合同编号、合同类别（3 列）

3.6 校验规则


字段类型规则不通过标记
邮箱正则匹配待核对邮箱
日期YYYY-MM-DD，年份 1900-2100日期格式异常
电话去符号后数字 ≥ 8 位电话格式可疑
金额纯数字，最多一个小数点金额待清理
年龄0-120 整数年龄超出范围
性别男/女/未知期望值不符
合同期限类型有固定期限/无固定期限期望值不符

3.7 校对界面


  • 左侧：PDF 原版（Canvas 逐页渲染，上下滑动），支持翻译切换
  • 右侧：字段表格（提取值 / 修正值 / 校验状态），点击字段值左侧高亮定位
  • 翻译：LLM 译为中文，日期/金额/人名/邮箱等保留原文，结果缓存
  • 重提：重新提取不丢失手动修正值，自动清翻译缓存

3.8 导出


  • 格式：Excel (.xlsx)，46 列
  • 范围：选中文件 / 全部已处理 / 仅失败文件
  • 空值：显示为空白单元格
  • 校验警告：黄色高亮 + 批注
  • 失败文件：自动填入员工编号、合同编号（推算）、错误信息

3.9 配置


配置项说明
LLM 服务商DeepSeek / 智谱 AI
API Key本地存储，脱敏显示
模型名称默认 `deepseek-chat`
OCR阿里云（AccessKey），可选不配

4. 异常处理


场景处理
非 PDF 文件跳过
PDF 加密弹窗输入密码
文字层为空 + OCR 未配置标记失败
LLM 返回非 JSON重试，仍失败则标记失败
网络中断自动重试
重复导入MD5 去重

5. 非功能需求


  • 性能：单文件 ≤ 20 秒，3 文件并发
  • 安全：所有密钥仅存本地，不上传
  • 兼容：Windows 10/11，分辨率 ≥ 1366×768
  • 维护：字段配置化（JSON），代码模块化

合同智能提取工具 — 使用说明


版本：V1.7 | 更新日期：2026-06-18

一、软件概述


本工具用于批量处理 PDF 合同文件，自动提取员工信息、薪酬、银行账户等 45 个关键字段，支持人工校对后导出为 Excel。
适用文件：劳动合同、调薪文件（PDF 格式，支持文字型和扫描件）

二、安装与启动


环境要求

  • Windows 10 / 11 操作系统
  • 无需额外安装运行环境（软件自带）

启动步骤

1. 双击程序图标启动 2. 等待主界面加载完成

三、首次使用 — 配置


首次使用需要配置大模型 API Key（必须）和阿里云 OCR（可选，仅扫描件需要）。

3.1 大模型配置


点击左侧菜单 设置，在"大模型配置"卡片中填写：
配置项说明示例
大模型服务商选择 DeepSeek 或 智谱 AIDeepSeek
模型名称保持默认或填入模型代号`deepseek-chat`
API Key服务商提供的密钥`sk-xxxxxxxx`

> 获取 API Key： > - DeepSeek：访问 https://platform.deepseek.com → 注册 → API Keys → 创建 > - 智谱 AI：访问 https://open.bigmodel.cn → 注册 → API Keys → 创建
填写后点击 测试连接，显示"连接测试成功"即配置完成。点击 保存。

3.2 阿里云 OCR 配置（可选）


如果处理的是扫描件 PDF（文字层为空，需要 OCR 识别），需要配置阿里云 OCR：
配置项说明
AccessKey ID阿里云 RAM 用户的 AccessKey
AccessKey Secret对应的 Secret
区域默认 `cn-hangzhou`

> 获取 AccessKey：登录阿里云控制台 → RAM 访问控制 → 用户 → 创建用户 → 勾选"OpenAPI 调用访问" → 获取 AccessKey ID/Secret → 为该用户授权 AliyunOCRFullAccess 权限。
填写后点击 测试连接，点击 保存。

1. 勾选需要处理的文件（或点击 开始处理 自动处理所有"待处理"文件） 2. 点击 开始处理 按钮 3. 顶部显示处理进度（完成数 / 失败数 / 当前步骤）
每次最多 3 个文件同时处理，失败会自动重试 1 次。

遇到加密 PDF 时，会自动弹出密码输入框。输入密码后点击 确定，错误则提示重试。不知道密码可点击 跳过，该文件标记为失败。

九、常见问题


Q1：为什么有些字段是空的？

LLM 从 PDF 原文中找到对应信息才会填入。如果原文中确实没有该信息（如没有 IBAN），字段留空是正常的。请在校对面板中人工补充。

Q2：扫描件处理不了怎么办？

确认为扫描件（文字层为空）时，需要： 1. 在设置页配置阿里云 OCR AccessKey 2. 重新处理该文件，工具会自动调用 OCR 识别

Q3：调薪文件的字段为什么是空的？

调薪文件和劳动合同使用同一套 42 字段提取。调薪文件通常只包含薪酬变更信息，其他不相关的字段（如个人信息）在原文中不存在，自然会留空。如需对这些字段补充，可在校对面板手动编辑，或点击 重新提取 按钮重试。

Q4：PDF 加密了怎么办？

处理到加密 PDF 时，会自动弹出密码输入框，输入密码即可。如果不清楚密码，点击"跳过"。

Q5：API Key 在哪里获取？

  • DeepSeek：https://platform.deepseek.com → 注册 → API Keys
  • 智谱 AI：https://open.bigmodel.cn → 注册 → API Keys
  • 阿里云 OCR：阿里云控制台 → RAM 访问控制 → 创建 AccessKey

Q6：处理失败的文件怎么处理？

点击 原件 按钮查看原文件，确认文件是否损坏、是否为支持的合同类型。然后在文件管理页面重新勾选 → 开始处理 重试。

Q7：OCR 失败但字段提取正常？

这是正常现象——文件本身是文字型 PDF，OCR 虽失败但 pdfjs 回退拿到了文字，LLM 照样能提取字段。状态标绿色"已完成"即可，翻译也正常工作。

Q8：文件重复导入怎么办？

点击文件管理页右上角的 清理重复 按钮，自动按 MD5 去重，保留提取字段最多的那条。
